# codyssey_b1_3

코딩 없이 마우스 클릭만으로 일이 자동으로 돌아가게 만들기

## 🏆 [프로젝트 1] 자동화 도구 비교 구현: GitHub 이슈 관리 워크플로우

### 1. 사용한 도구 이름

- **Make** (구 Integromat)
- **[Relay.app](http://Relay.app)**

### 2. 구현 과정 요약

- **공통 워크플로우:** GitHub에 이슈(Issue)가 생성되면 상세 정보를 가져온 후, '버그' 관련 이슈는 [긴급_버그 트래커] 시트에, 그 외 이슈는 [전체_이슈 백업] 시트에 자동으로 분류/적재하는 파이프라인을 구축함.
- **Make 구현:** `Watch Issues` 트리거와 `Get an Issue` 모듈을 연동하여 배열 데이터를 확보하고, `Router`를 통해 키워드 필터링(버그) 및 `Fallback` 경로(일반 이슈)를 설계함.
- **[Relay.app](http://Relay.app) 구현:** `New Issue` 트리거를 활용하여 선형적인 `Path(If/Else)`를 생성하고, 조건문을 통해 각 시트로 데이터를 배분하는 직관적 워크플로우를 구현함.

### 3. 비교 분석 (5개 항목 이상)


|               |                           |                                   |
| ------------- | ------------------------- | --------------------------------- |
| **비교 항목**     | **Make**                  | **[Relay.app](http://Relay.app)** |
| **UI/UX**     | 노드 중심의 2D 캔버스, 자유로운 설계 가능 | 선형(Linear) 구조의 깔끔하고 직관적인 UI       |
| **설정 난이도**    | 데이터 구조(배열/객체) 이해 필요 (상)   | 변수 매핑이 매우 쉽고 직관적 (하)              |
| **연동 서비스 범위** | 1,500개 이상의 방대한 앱 커넥터 제공   | 핵심 비즈니스 앱 위주의 엄선된 연동              |

### 4. 도구별 장단점 정리

- **Make**
  - **장점:** 정교한 데이터 가공(`map`, `join`, `regex`)이 가능하며, 에러 핸들링을 세밀하게 설계할 수 있음.
  - **단점:** 초기 학습 곡선이 높고 데이터 구조(JSON/Array)에 대한 이해가 필수적임.
- **[Relay.app](http://Relay.app)**
  - **장점:** 별도의 함수 학습 없이 즉시 자동화 구축이 가능하며, 승인(Approval) 프로세스 등 협업 기능이 강력함.
  - **단점:** 복잡한 API 데이터 변환이나 정밀한 파싱 작업에는 다소 제한적임.

### 5. 적합한 상황에 대한 의견

- **Make는 다음과 같은 상황에 적합합니다:**
  - 다수의 외부 API를 복합적으로 연동하고 데이터를 정교하게 가공해야 하는 복잡한 백엔드 자동화.
  - 특정 모듈에서 에러 발생 시 재시도 전략 등을 상세히 설계해야 하는 비즈니스 핵심 프로세스.
- **[Relay.app](http://Relay.app)은 다음과 같은 상황에 적합합니다:**
  - 담당자의 승인이나 확인이 필요한 결재선/협업 위주의 업무 자동화.
  - 코딩 지식이 없는 실무자가 10분 내외로 빠르게 워크플로우를 구축해야 하는 서비스 운영 업무.

# [Codyssey B1-3] 프로젝트 2: 실시간 기상 재난 자동 전파 시스템

## 1. 자동화할 반복 업무 정의

- **업무명:** 실시간 기상 관측 데이터를 활용한 폭염/호우/강풍 긴급 자동 경보 시스템
- **정의:** 1분 단위로 유입되는 기상 관측 데이터(지점 온도, 강수량, 풍속)를 모니터링하여, 재난 위험 임계치 도달 시 AI가 생성한 경보 문구를 포함한 이메일을 유관 부서에 즉시 자동 발송하는 작업.

## 2. 자동화 도구 선정 및 이유

- **선정 도구:** **[Relay.app](http://Relay.app)**
- **선정 이유:**
  - **실시간 데이터 처리:** 시트의 실시간 변화(New Row)를 트리거로 즉각 감지 가능.
  - **직관적인 자동화:** 복잡한 코딩 없이 필터링과 액션을 선형적으로 연결하여 유지보수가 용이함.
  - **통합 용이성:** 별도의 승인 절차 없는 완전 자동화 흐름을 구성할 때, 시스템 결함 없는 깔끔한 액션 연결이 가능함.

## 3. 워크플로우 설계 문서 (다이어그램)

### 워크플로우 흐름 설명

1. **Trigger (New Row Added):** 기상 관측 데이터가 Google Sheets에 실시간으로 입력됨.
2. **Filter (조건 분기):** 데이터가 '재난 임계치(폭염 33도, 호우 0.5mm, 강풍 5.0m/s)'를 초과하는지 판단.
3. **Action 1 (AI 요약):** 임계치 초과 시, OpenAI 모듈이 유입된 Raw 데이터를 분석하여 상황별 행동 요령이 포함된 문구 생성.
4. **Action 2 (최종 자동 전파):** 생성된 메시지를 Gmail을 통해 유관 부서에 즉시 자동 발송.
5. **Error Handling (예외 처리):** 실패 시, [임시_데이터_적재_시트]에 에러 내용과 데이터를 즉시 백업.

## 4. 구현 상세 (프로젝트 요구사항 충족)

- **보너스 1 (AI 연동):** OpenAI Action을 통해 단순 수치를 "현재 기온 {{TA}}도로 매우 위험한 수준입니다. 야외 활동을 즉시 중단하세요."와 같은 실질적인 정보성 메시지로 자동 변환함.
- **보너스 2 (실패 전략):** Gmail 발송 실패 시, 에러가 발생한 이슈 정보를 별도의 시트([임시_데이터_적재_시트])로 즉시 자동 적재하는 대체 경로 설계(Fail-safe).

## 5. 보고서용 워크플로우 구현 상세 및 결과 요약

### [구현 단계 캡처 가이드]

- **구현 화면:** [Relay.app](http://Relay.app) 내의 `Trigger` -> `Filter` -> `OpenAI` -> `Gmail`로 이어지는 깔끔한 선형 노드 흐름을 캡처하십시오.
- **실행 결과:** 시트에 재난 데이터를 수동으로 입력한 후, 1분 내에 내 이메일로 AI가 작성한 경보 메일이 자동으로 도착한 화면을 캡처하십시오.
