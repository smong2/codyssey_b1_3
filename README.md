# codyssey_b1_3

코딩 없이 마우스 클릭만으로 일이 자동으로 돌아가게 만들기



## 🏆 [프로젝트 1] 자동화 도구 비교 구현: GitHub 이슈 관리 워크플로우

### 1. 사용한 도구 이름

- **Make** (구 Integromat)
- **[Relay.app](http://Relay.app)**

### 2. 구현 과정 요약

- **공통 워크플로우:** GitHub에 이슈(Issue)가 생성되면 상세 정보를 가져온 후, '버그' 관련 이슈는 [긴급_버그 트래커] 시트에, 그 외 이슈는 [전체_이슈 백업] 시트에 자동으로 분류/적재하는 파이프라인을 구축함.
- **Make 구현:** `Watch Issues` 트리거와 `Get an Issue` 모듈을 연동하여 배열 데이터를 확보하고, `Router`를 통해 키워드 필터링(버그) 및 `Fallback` 경로(일반 이슈)를 설계함.
- **[Relay.app](http://Relay.app) 구현:** `New Issue` 트리거를 활용하여 선형적인 `Path(If/Else)`를 생성하고, 조건문을 통해 각 시트로 데이터를 배분하는 직관적 워크플로우를 구현함.

### 3. 비교 분석 (5개 항목 이상)


|               |                           |                                   |

### 4. 도구별 장단점 정리

- **Make**
  - **장점:** 정교한 데이터 가공(`map`, `join`, `regex`)이 가능하며, 에러 핸들링을 세밀하게 설계할 수 있음.
  - **단점:** 초기 학습 곡선이 높고 데이터 구조(JSON/Array)에 대한 이해가 필수적임.
- **[Relay.app](http://Relay.app)**
  - **장점:** 별도의 함수 학습 없이 즉시 자동화 구축이 가능하며, 승인(Approval) 프로세스 등 협업 기능이 강력함.
  - **단점:** 복잡한 API 데이터 변환이나 정밀한 파싱 작업에는 다소 제한적임.

### 5. 적합한 상황에 대한 의견

- **Make는 다음과 같은 상황에 적합합니다:**
  - 다수의 외부 API를 복합적으로 연동하고 데이터를 정교하게 가공해야 하는 복잡한 백엔드 자동화.
  - 특정 모듈에서 에러 발생 시 재시도 전략 등을 상세히 설계해야 하는 비즈니스 핵심 프로세스.
- **[Relay.app](http://Relay.app)은 다음과 같은 상황에 적합합니다:**
  - 담당자의 승인이나 확인이 필요한 결재선/협업 위주의 업무 자동화.
  - 코딩 지식이 없는 실무자가 10분 내외로 빠르게 워크플로우를 구축해야 하는 서비스 운영 업무.
