1단계: 프로젝트 기획 및 환경 세팅
무작정 툴을 켜기 전에, 워크플로우의 데이터 흐름(Data Flow)을 먼저 정의해야 합니다.

P1(비교 프로젝트) 워크플로우 확정: 가장 범용적이고 테스트가 쉬운 조합(예: Google Forms/Sheets → 조건 분기 → 이메일/Slack/Discord)으로 데이터 구조를 기획합니다.

P2(자유 주제) 워크플로우 발굴: 평소 반복적으로 확인하거나 옮겨 적는 데이터(예: 특정 조건의 알림 자동화, 주문/에러 로그 수집 등) 중 하나를 선정합니다.

테스트용 계정/채널 생성: 권한 문제나 기존 데이터 오염을 막기 위해, 테스트용 구글 시트 파일과 메신저(Slack/Discord) 채널을 별도로 생성합니다.

2단계: [프로젝트 1] 비교 분석 구현 및 문서화
동일한 로직을 두 개의 다른 도구(예: Zapier, Make)에서 구현하고 차이점을 도출합니다.

기준 툴(예: Zapier) 로직 구현: * Trigger 세팅 (예: 시트에 새 행 추가됨)

Filter/Router 적용 (예: 특정 열의 값이 '긴급'인 경우)

Action 2개 연결 (예: 메신저 알림 발송 + 다른 시트에 이력 기록)

분기(Branch) 테스트: 라우터에 설정한 조건들이 모두 정상 작동하는지, 각 경로가 최소 1회 이상 실행되도록 테스트 데이터를 주입하여 확인합니다. (실행 화면 캡처 필수)

대조 툴(예: Make) 복제 구현: 동일한 로직을 다른 툴에서 구현합니다. 이때 각 노드(Node) 간 데이터 매핑 방식의 차이를 유심히 관찰합니다.

Markdown 보고서 작성: UI/UX, 분기 설정의 직관성, 무료 플랜 한도 등을 비교하는 표를 작성하고, 개발 및 운영 관점에서의 장단점을 정리합니다.

3단계: [프로젝트 2] 자유 주제 설계 및 자동화 파이프라인 구축
선택한 1개의 도구로 완전히 자동화된(Trigger 발생 시 즉각 동작하는) 파이프라인을 완성합니다.

도구 선정 이유 작성: P1에서 파악한 각 툴의 특성을 근거로, 현재 구축하려는 시스템의 스펙(데이터 처리량, 복잡도 등)에 더 적합한 툴을 하나 선정합니다.

자동화 파이프라인 구현: 기획했던 자유 주제의 Trigger, 분기 로직, Action 노드들을 연결합니다.

설계 문서 작성: 다이어그램이나 Markdown을 활용하여 데이터의 흐름과 조건 분기의 기준을 명확히 문서화합니다.

4단계: 보너스 과제 적용 (고도화)
요구사항을 모두 충족했다면, 시스템의 가치와 안정성을 높이는 작업에 들어갑니다.

AI 프롬프트 엔지니어링 (보너스 1): 워크플로우 중간에 ChatGPT나 Claude 노드를 추가합니다. 단순 텍스트 전달을 넘어, 디테일한 페르소나와 출력 형식을 지정하는 정교한 프롬프트를 삽입하여 요약본이나 템플릿 메시지의 퀄리티를 높여봅니다.

예외 처리 및 에러 핸들링 (보너스 2): 특정 노드(예: API 전송)에서 실패했을 때 실행될 'Error route'를 추가하여, 관리자 이메일이나 메신저로 실패 로그(Error message)를 쏴주는 로직을 구현합니다.

5단계: 최종 검증 및 보안 처리 (제출 준비)
가장 중요한 보안 및 마감 검토 단계입니다.

민감정보 마스킹 (필수): 캡처한 화면들에 API Key, 인증 토큰, 비밀번호, 개인 이메일 주소 등이 노출되지 않았는지 꼼꼼히 확인하고 모자이크 처리합니다.

무료 플랜 검증: 구현한 과정 중 유료 기능이 포함되었는지 확인하고, 포함되었다면 보고서에 불가피했던 이유를 명시합니다.

요구사항 체크리스트 대조: 1 Trigger, 2 Actions, 1 Filter/Router 조건이 두 프로젝트 모두에 잘 적용되었는지 최종 점검합니다.
